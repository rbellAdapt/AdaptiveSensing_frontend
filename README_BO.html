🏭 The AI Software Factory

Business Owner (@bo) Manual & Solution Manager Guide

Welcome to your AI-driven Agile Release Train (ART). You are no longer a solitary developer; you are the Business Owner and Solution Manager of a synthetic software factory. This environment is governed by strict roles, immutable file locks, and automated workflows. Your job is to inject Epics (high-level vision), tune the AI engines, and clear strategic roadblocks. Do not write code. Orchestrate.

👥 1. The Factory Setup (Agent Skills)

You interact with 8 specialized agents. Cross-contamination is strictly forbidden by the system's Write-Access Matrix.

The Leadership Triad (Your Direct Reports)

  • @prod (Product Manager): Focused on R&D, academic literature synthesis, and value streams.
  • @arch (System Architect): Translates vision into blueprints. The only agent allowed to write to docs/implementation_plan.md and docs/session_history.md.
  • @rte (Release Train Engineer): An invisible background daemon. Acts as the Scrum Master to enforce rules and handoffs. Will only speak to you if the system deadlocks.

The Execution Team (The Factory Floor)

  • @dev (Lead Developer): Writes the actual code. Locked strictly to the src/ directory.
  • @sme (Subject Matter Expert): The scientific/math consultant. Verifies algorithmic fidelity and physics logic. Read-only.
  • @qa (QA Engineer): Writes test suites. Locked strictly to the tests/ directory.
  • @brand (Brand Specialist): Audits UI, UX, and copy. Read-only.
  • @ops (DevSecOps): Manages infrastructure, security, .gitignore payload hygiene, and Git terminal executions.

📜 2. The Core Rules & Protocols

The factory operates efficiently because the AI is bound by unbending rules (stored in .agents/rules/):

  • The Write-Access Matrix: Agents physically cannot modify files outside their domain. @dev cannot fix tests; @qa cannot fix code.
  • The Silent Chain-of-Thought (SCoT) Log: Agents must write their step-by-step reasoning to docs/logs/thought_trace.md before coding. They are physically barred from reading it back to prevent memory bloat. If you don't understand an AI's decision, read its mind here.
  • The Two-Strike Escalation: If @dev fails an audit from @sme, @qa, or @brand twice, @rte pauses the system and asks @bo for a tie-breaking decision to prevent infinite API billing loops.

⚡ 3. The /dir Workflow (Direct Execution)

In a heavy Agile pipeline, waking up @arch, @qa, and @rte to change a button color or run a simple Python script wastes massive compute. You can bypass the entire bureaucracy instantly using the /dir slash command.

What it does: It triggers the direct-execution.md workflow. It forces all non-targeted agents to go to sleep, disables blueprinting, and grants the targeted agent temporary autonomy to execute your command immediately.

Auto-Resume: The moment your next prompt doesn't contain /dir, the full Agile factory automatically wakes back up.

Example Scenarios for /dir:

  • Cosmetic Fast-Track: "/dir @dev and @brand: Update header.tsx padding to p-4 and do a quick Vibe Check."
  • Ops Hotfix: "/dir @ops: The Nginx container is failing. Change the port mapping in docker-compose from 80 to 8080 and push the fix."
  • Academic Brainstorm: "/dir @prod and @sme: Let's discuss the viability of using a Monte Carlo simulation for this next Epic. Just give me your thoughts, no blueprints yet."
  • Micro-Script: "/dir @dev: Write a standalone 15-line Python script in the root directory to parse this CSV. Do not touch src/."

🗣️ 4. Operating the Factory (Standard Usage)

When you aren't using /dir for micro-tasks, you use the standard Chain of Command to build major features.

🔄 5. Automated Workflows (Antigravity Syncs)

Your workspace includes three atomic Git workflows (stored in .agents/workflows/). You trigger these by typing their names into the chat.

  • session-end-atomic-git.md: Commands @arch to compress the session memory, write a handover note, and commands @ops to commit and push the repo safely.
  • session-restart-atomic-git.md: Commands @ops to pull the repo, and @arch to ingest the handover note and re-orient the team.
  • atomic-commit-push.md: A rapid save-state triggered autonomously or manually during deep coding sessions.

🧠 6. The Engine Strategy (Fast vs. Thinking)

Do not leave your AI stuck in one mode. You act as the manual transmission, shifting gears based on the phase of your Agile sprint.

  • Sprint Planning = Thinking Mode: When generating the Epic, writing the .plan, or verifying heavy physics with @sme, use a reasoning model. The AI will think deeply before typing.
  • Sprint Execution = Fast Mode: Once the .plan is saved, toggle your IDE to a standard/fast model. Tell @dev to execute the plan. It will blitz through code generation at lightning speed.
  • Deadlock Resolution = Thinking Mode: If @rte pauses the train due to a bug, switch back to Thinking Mode to analyze the SCoT log and break the tie.

💼 7. The Business & Model Selection Strategy

You pay for Google Workspace and have access to the Gemini Ultra/Pro API. You do not need to pay for monthly subscriptions to run this factory.

Primary Engine: Gemini (1.5 Pro / 2.0 Pro)

Run 95% of your factory on Gemini.

  • Why: Gemini has a massive 2-million token context window. In scientific coding, you can load dozens of academic PDFs, textbooks, and your entire codebase into the memory, and Gemini will synthesize it flawlessly.
  • Cost: Highly cost-effective since it aligns with your existing Google ecosystem.

The "Sniper" Insurance: Claude / OpenAI (Pay-as-you-go API)

Do not buy a second subscription. Instead, create a free API account with Anthropic (Claude) or OpenAI and load exactly $10 in API credits.

  • Why: AI models have specific specialties. Claude 3.5 Sonnet is the undisputed king of React/UI coding. OpenAI o1/o3-mini is incredible at mathematical deadlocks.
  • How to use: If Gemini's @dev gets stuck on a stubborn frontend bug and hits the Two-Strike rule, simply toggle your IDE's API dropdown to Claude, say "@dev, fix this UI bug," and let Claude write the code. Once fixed, toggle back to Gemini. This costs fractions of a cent per use and prevents you from ever getting permanently stuck.
